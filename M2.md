## 环境变量

**本地**：`.env.local`（**不得提交 Git**）。

| 变量 | M2 | 说明 |
|------|---|------|
| `APP_PASSCODE_HASH` | **必填（生产）；本地可配合文档生成** | bcrypt / argon2 哈希全串 |
| `SESSION_SECRET` | **必填** | HMAC/API 签发依赖；足够长随机 |
| `AIRTABLE_PAT` | 可选 | M2 不接业务时可占位或省略 |
| `AIRTABLE_BASE_ID` | 可选 | 同上 |
| `AIRTABLE_MATERIALS_TABLE` | 可选 | 同上 |
| `AIRTABLE_CONSUMPTION_TABLE` | 可选 | 同上 |

校验规则建议写在 `lib/env.ts`：**生产**缺失 `SESSION_SECRET`、`APP_PASSCODE_HASH` → 报错。

**不要使用**明文 `APP_PASSCODE` **做生产校验**。

## Shared passcode 登录

**流程**

1. 用户于 `/login` 输入口令 → `POST /api/login`（JSON `{ "passcode": "..." }` 或表单字段皆可，团队统一）。
2. 服务端 `bcrypt.compare` / argon2 `verify` 对比 `APP_PASSCODE_HASH`。
3. 成功：设置 **httpOnly** cookie  
   - `Max-Age`：**180 天或 365 天**（与产品约定一致）  
   - **Production**：`Secure=true`、`SameSite=Lax`  
4. 失败：401，前端展示错误文案（不明确提示「密码错误」与「限速」混同即可，或可区分）。

**Cookie 必须可验证** — 禁止仅写入随机不透明字符串且无签名校验：

- **推荐**：`jose`（HMAC JWT，payload 含 `exp` / `iat`，可选 claim `authenticated`、`sessionVersion`）  
- **或**：`iron-session` 密封会话  

**Payload 示意**（JWT claims 或与 iron-session store 等价）：

```json
{
  "authenticated": true,
  "sessionVersion": 1,
  "iat": 1730000000,
  "exp": 1761536000
}
```

轮换 **`SESSION_SECRET`** 或 **提升 `sessionVersion`**（校验时比对）可用于「设备遗失 / 全员下线」——与 [plan-airtable.md](plan-airtable.md) 一致。

**`POST /api/logout`**

- [ ] 清除会话 cookie（过期或清空 payload），前端跳转 `/login`。

## 登录限速

- **目标**：`/api/login` **按 IP**，**滑动或固定窗口**（例如 **10 次 / 10 分钟**），超限 **429**。  
- **MVP**：`lib/rate-limit.ts` 内存 `Map`。  
- **说明**：Serverless **多实例不共享内存** — 记录在文中，MVP 可接受；升级路径：Upstash Redis。

---

## API 门禁

- **`lib/auth.ts`**：  
  - `requireAuth(request):` 无/无效会话 → 抛错误或返回 `NextResponse.json(..., { status: 401 })`（与路由统一风格）。  
  - 已从 cookie 校验通过则继续。

**M2 Protected 示例**：`GET /api/health`

| 场景 | HTTP |
|------|------|
| 未登录或无有效 cookie | **401** |
| 已登录 | **200**（body 可为 `{ ok: true }`） |

**`POST /api/login`、`POST /api/logout`**：**不**走 `requireAuth`。

将来 `GET /api/materials`、`POST /api/consumption` 等一律复用 **`requireAuth`**。
