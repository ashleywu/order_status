# Milestone 4 — 采集主流程 UI（Mock 提交）

对齐：[M3.md](M3.md)（真实物料数据源 + UI 契约）、[plan-airtable.md](plan-airtable.md) §4。**本阶段不接** `POST /api/consumption`、不写 Airtable 消耗表；提交为 **mock**，真实记账见后续 **M5**。

---

## 目标

在已能 **安全读取并缓存**物料的前提下，完整打通手机端 **核心采集 UX**：

**选物料 → 选用途 → 调数量 → 确认页雏形**，且 **可先接 M3 真数据**，**提交动作为 mock**。真正写入、`clientRequestId`、Undo、离线 → **M5**。

---

## 原则

- **无搜索框**
- **不依赖软键盘**
- **少滑动**，**大按钮**
- **单手 / 手头不便**可操作
- **UI 只服务消耗采集**，不做后台管理

## 2. 用户流程

### 主流程状态机（概念）

```
material → usage_type → quantity → review → success (mock)
```

### 选物料后的统一行为（**钉死**）

- **从 Home 高频区或 `/pick` 点选某物料** → **直接进入 `/usage`（选用途）**，**少一步**。
- **`/usage` 顶栏**：展示 **当前物料摘要**（`buttonLabel || name` + `unit` 等）+ **`Change` /「更换物料」** → 回到 **`/pick` 或 Home**（按实现择一，全库统一）；**Change 视为重选物料**（见 §8 **状态清理**）。

### 自由度

用户在任一步均可：

- **系统返回 / 返回上一级**（路由栈自然弹出；单页 stepper 则等价 `history.back`）  
- **重置本次记录** → 清空 `DraftConsumption` → **`/`**  

### 成功后

Mock 成功后：**最近使用**写 `localStorage`；清空 **draft**；回 **Home** 或展示 **「再记一笔」** → 回 Home。

## 3. Home

**目标**：高频 **一屏可点**；**禁止**把首页拉成无限长列表。

| 模块 | 要求 |
|------|------|
| 高频区 | `favorite === true`；按 `sortOrder`（再 `name`）；**首页最多展示 12 条** |
| 超出 | 显示 **「View all」**（或等价文案）→ 导航至 **`/pick`**（Material Picker） |
| 最近使用 | **可选**；规则见 **§7 / §8**（**最多 8 条**、prepend、去重） |
| 分类入口 | Ingredient / Bottle / Label / Ribbon / Packaging → 可深链到 `/pick?category=…`（可选） |
| 刷新 | **继承 [M3.md](M3.md)**：大号刷新、`loading disabled`、错误可见 |

### 验收

- 点高频物料 **直接进入 `/usage`**（见 §2）  
- **无搜索框**；主点击目标足够大  
- **物料为空** → **M3 空状态**  
- 首页 **不出现**第 13 个 favorite（须 **View all**）

## 4. Material Picker

**目标**：不靠键盘能找到物料。

| 模块 | 规则 |
|------|------|
| 结构 | **Category tabs** + **Group chips**（仅限 **当前分类**下有数据的 `group`）+ **大按钮 grid** |
| 数据 | 仅 **`active`**（M3/API 已滤） |
| 默认 tab | **`favorite` 最多的分类** / **上次使用分类** / **`ingredient`** — 选一写死在产品；并列时 **`category` 按固定优先级**（例如 `plan-airtable.md` §2 所列顺序）兜底 |
| 按钮文案 | **`buttonLabel \|\| name`** + **轻量单位** |
| 视觉 | 可选 category 色条/图标；**不得 hover-only** |
| 排序 | `favorite` desc → `sortOrder` asc → `name` asc |

### 控件规格建议

| 项 | 值 |
|----|-----|
| 最小行高（按钮） | **≥ 64px**（主推） |
| 触控靶 | **≥ 44×44 px** |
| 长文案 | **最多两行 + ellipsis** |

### 验收

分类 + 分组可找到示例物料；**无软键盘**；切 tab/chip **不卡顿**；长名 **不破版**

## 5. Usage Type（`/usage`）

**目标**：四类用途 **巨大**、不可误解；**已选物料**在顶栏一眼可见。

| UI | `usage_type` 值 |
|----|----------------|

## 6. Quantity（`/quantity`）— **全日整数（钉死）**

**目标**：不用输入框完成常见数量。**M4 阶段数量域仅允许整数。**

| 规则 | 说明 |
|------|------|
| **整数** | `defaultIncrement`、`quantity`、**所有 `quickAmounts`**、以及 **`+/−` 的结果**均为 **整数** |
| **步进网格（与 M3 一致）** | 任意合法 **`quantity` 必须为 `defaultIncrement` 的正整数倍**（`quantity % defaultIncrement === 0`）；**初值、`+/−`、quick 跳转**不得落在非倍数上，否则 **`−`** 会破坏「最小值为 `defaultIncrement`」语义 |
| **`+/−`** | 每次 **`± defaultIncrement`**（整数） |
| **初值** | 初次进入 Quantity：**`quantity = defaultIncrement`**（整数）。**若由「重选用途」进入**：同 **§8** 表 — **重置为 `defaultIncrement`** |
| **下限** | **最小值 = `defaultIncrement`**（与步长相等）；触底时 **`−` disabled** |
| **未来小数**（非 M4） | 例如 **0.5 ml** → **只允许**通过 Airtable / 服务端 **`quickAmounts` DTO** 下发 **离散**按钮（如 `[0.5,1,2,5]`）；**禁止**键盘输入 |

### Quick amounts（与 M3 **同一模块**）

- **数据源**：优先物料 DTO **`quickAmounts`**（`GET /api/materials` 已解析为 **整数数组**）。  
- **若为空**：客户端调用 **`lib/quantity-defaults.ts`** 与 **M3 服务端**一致的 **`defaultQuickIntsForUnit(unit)`**（或同名函数）；**不得在 M3 route 与 M4 UI 各写一遍魔法数字**。  
- **与 M3「步进网格」一致**：Chip 对应数量 **必须**为 **`defaultIncrement` 的正整数倍**（且 **≥ defaultIncrement**）。若缓存或残缺 DTO 出现 **非倍数**或 **更小值**：以 **`lib/quantity-defaults.ts`** 与 **[M3.md](M3.md)** **同一策略**校正（剔除或提升至最小合法倍数），**禁止**在 UI 上展示会破坏 **`±`/下限**规则的数字。  

### Airtable 原始串

- 服务端解析 **`"1,5,10"`** 时：**只保留整数**；非整数 token **丢弃**（若业务将来要 0.5，须改 DTO 策略而非 M4 键盘）。

### 验收

全靠大按钮；**无输入框**；**无 0 / 负 / 非整数**；数量与单位醒目

## 7. Review / Mock Submit

**目标**：真实 API 前就定稿 **提交前后体验**。

| Review 摘要 | `buttonLabel/name`、`usage`、`quantity + unit`、**本地化当前时间** |
|---------------|----------------------------------------------------------------------|
| 操作 | **大号 Submit**、`返回修改`（回到 **`/quantity`**；**不改变** §8 中 `materialId`/`usageType`） |

### Draft 在进入 Review/Mock 前须有效（钉死）

- **`draft.materialId` 须在**当前 **`materials` map**中存在；若在向导中途静默刷新后被 **下架 / 停用**，须在 **跳转 `/review` 前或按 Submit 瞬间**发现并进入 **可读错误**：「该物料已不可用」+ **回 `/pick`** 或 `/`，勿 **mock success + 写入最近使用**。  
- **直达 URL**：`/usage`、`/quantity`、`/review` 在未满足前置字段时 **`replace`→ `/` 或 `/pick`**，禁止半截空白页。

### Mock 行为

1. **`loading` 300–800ms（随机或固定均可）**  
2. **`success` 全屏或大幅面**反馈  
3. **最近使用**（Mock 成功后 **钉死**）：  
   - **localStorage**：`entries[]`，每项含 **`materialId`**、**`usedAt`**（ISO）；新记录 **prepend**，同 id 只保留最新 **`usedAt`**，总长 **≤8** 删最旧。  
   - **展示**：列表只持久化 id + 时间；**名称**从当前内存 **materials map**（M3 缓存）读取；暂无则占位。  
4. **`DraftConsumption` 清空**  
5. **回到 `/` 或「再记一笔」**

### M4 **不做**

Airtable 写、`clientRequestId`、`Undo`

### 「返回修改」与 §8

- **Review → 返回改数量**：`/quantity`，**保留** `materialId` + `usageType`。  
- 其它拆分见 §8 表。

### 验收

| # | 项 |
|---|-----|
| 1 | End-to-end 可走通 |
| 2 | Mock 后最近使用 **prepend、去重、≤8** |
| 3 | 「返回修改」与 §8 **不丢不该丢** |
| 4 | 提交中 **Submit disabled / loading** |

## 8. 状态管理、清理规则与 401

### 草稿类型

```ts
type DraftConsumption = {
  materialId?: string
  usageType?: 'rd_lab' | 'product' | 'marketing' | 'workshop'
  quantity?: number
}
```

### 路由间清理规则（**钉死**）

| 用户动作 | `materialId` | `usageType` | `quantity` |
|----------|----------------|-------------|------------|
| **重选物料**（Home/Picker/`Change`） | **新值** | **清空** | **清空** |
| **重选用途**（回到 `/usage` 改选） | 保留 | **新值** | **重置为该物料默认值**：`quantity = defaultIncrement`（整数） |
| **从 Quantity 顶部返回 Usage**（未改物料） | 保留 | 保留 | 保留 |
| **Review → 返回改数量** | 保留 | 保留 | 保留 |
| **重置本次** | **清空** | **清空** | **清空** |

> 从而避免「Bottle 的用途/数量误套到 Ingredient」一类脏状态。

### 401（流程半途会话失效）（**钉死**）

任一依赖 `GET /api/materials` 的请求若得 **`401`**（含静默刷新、手动刷新、`/pick` 重进）：

1. **`DraftConsumption`** 与 **`sessionStorage`（若有）须在**同一 401 handler **内一齐清空**，避免登录页返回后 hydrate 半截向导。  
2. **物料缓存**标记 **`stale`**（或：**删除**物料 `localStorage` 包 — 二者择一并在实现内一致）；**严禁**静默展示「看起来像仍登录」的假列表而不提示。  
3. **跳转 `/login`**（与 [M3.md](M3.md) UI 契约一致）。  
4. **登录成功** → **强制重新拉** `GET /api/materials` 再让用户从 **`/`** 开始。

### 持久化

| 数据 | 位置 |
|------|------|
| 最近使用 | `localStorage`，结构见 §7 |
| 物料快照 | 见 [M3.md](M3.md) 物料 `localStorage` 策略 |
| Draft | **`sessionStorage` 可选**；**刷新可丢草稿**（不提示也行） |

### 原则补充

- **不承诺离线提交** · **不写 Airtable**

**推荐路径**（`draft` 放 **client store** 或 `sessionStorage`，与 URL query 可自行取舍）：

| Path | 屏 |
|------|-----|
