# Milestone 3 — 读取物料 + 缓存刷新

对齐总计划：[plan-airtable.md](plan-airtable.md) §5.5–§6；前置：[M1.md](M1.md)（Base）、[M2.md](M2.md)（会话与门禁）。

响应 JSON 字段名与 **[plan-airtable.md](plan-airtable.md) §6.1 `materials[]`** 保持一致（下文示例）。

---

## 目标

让手机端能 **在不经过浏览器直连 Airtable** 的前提下，**安全、稳定**拿到可用于 **大按钮列表**的 **active materials**：

- **M3 完成后**：手机上 **还不能提交消耗**，但已是 **真实 Airtable** 数据，支持 **缓存、静默拉取、大号刷新**，并按 **`category` / `group`**（源自 `material_group`）与高亮 **favorite** 展示。
- **任何时刻**须有 **可读 UI 反馈**：**刷新、空列表、会话失效（401）、缺省字段展示**等均不得落成「空白/卡住」——详见 §「**UI 契约**」。
- 本里程碑的 **tabs / chips / grid / favorite 区块**可与 M4 **延续**，在此做成 **已可验收的雏形**。

## 服务端：仅 Route 接触 Airtable

**职责划分**

| 文件 | 用途 |
|------|------|
| `lib/airtable.ts`（或等价） | 封装带 PAT 的 `fetch`（list + 分页）；**仅**被 **`app/api/*/route.ts`**（或 Server Action）导入 |
| `lib/env.ts` | 已存在则扩展 PAT / Base / Table |

**原则**

- PAT **只**在 **server route**（及仅服务端 import 链）使用。
- **不**进入浏览器 bundle：**禁止** `NEXT_PUBLIC_*` PAT、禁止 Client 组件 import `lib/airtable.ts`。
- **不**从前端直接请求 `api.airtable.com`。

## `GET /api/materials`

| 项 | 要求 |
|----|------|
| 路由 | `app/api/materials/route.ts` |
| 鉴权 | `requireAuth()`；未登录 **`401`** |
| 数据源 | Materials 表；**仅 `active === true`**（`filterByFormula` 勾选语义以 Airtable 为准） |
| 排序 | **`favorite` 靠前** → **`sort_order` 升序**（缺失排后）→ **`name` 升序**。若 REST 不方便排序，则在 **服务端拉全页后排序**。 |
| 分页 | REST 单次最多 100 条：需 **`offset` 循环**直至无下一页 |
| 缺省与清洗 | **缺失字段**给合理默认（见下）；**无效 `category` / `unit`**：丢弃该行或记录在服务端日志（团队择一约定，验收前固定） |
| `quick_amounts` | 经 **`lib/quantity-defaults.ts`** 解析为 **`quickAmounts: number[]`**（**仅整数**）；逐项须为归一后的 **`defaultIncrement` 的正整数倍**（见下块）；串解析失败则走 `defaultQuickIntsForUnit` |
| `default_increment` | 同上模块规范为 **整数 ≥1** 的 **`defaultIncrement`**；与 quick 倍数规则共同决定 **`±`/chip** 自洽 |

### 严禁出现在响应中的内容

PAT、`supplier` 详情、price、notes、批次、`Material_batches`、后台规格大对象、`image` **完整 attachments 数组** 等——仅输出白名单（及单张 **`imageUrl`** 或 **`null`**）。

### 响应体形状（与白名单对齐）

顶层：

- `materials` —— 数组  
- `generatedAt` —— ISO 字符串  
- `schemaVersion` —— **整数常量**（如 `1`），契约/`localStorage` 形状变更时 **bump**  

`materials[]` **每一项**示例（可与 TypeScript `MaterialDto` / `MaterialsResponse` 一致）：

```json
{
  "materials": [
    {
      "id": "recXXXXXXXX",
      "name": "Lavender Essential Oil",
      "buttonLabel": "Lavender",
      "category": "ingredient",
      "group": "常用精油",
      "unit": "ml",
      "defaultIncrement": 1,
      "quickAmounts": [1, 5, 10],
      "favorite": true,
      "sortOrder": 10,
      "imageUrl": null
    }
  ],
  "generatedAt": "2026-05-26T12:34:56.789Z",
  "schemaVersion": 1
}
```

**映射说明**：Airtable 字段 **`material_group`** → JSON **`group`**；**`button_label`** → **`buttonLabel`**；**`default_increment`** → **`defaultIncrement`**；首张 attachment → **`imageUrl`** 或 **`null`**。

## 类型定义

建议单独文件便于前后端/import 类型共用（或由 `lib/materials-contract.ts` 导出）：

- `MaterialDto` —— 与白名单逐项一致  
- `MaterialsPayload` —— `{ materials; generatedAt; schemaVersion }`  

### `lib/quantity-defaults.ts`（M3 + M4 **共享**）

单一模块定义 **整数数量**的解析与兜底规则，**禁止**在 `app/api/materials/route.ts` 与 M4 UI 各写一份：

- 从 Airtable **`quick_amounts` 字符串**解析为 **`number[]`**：**只保留整数**（非整数 token 丢弃）。
- **`defaultQuickIntsForUnit(unit)`**（或同名）：当解析结果为空时，按 `unit` 分桶返回 **整数快捷数组**（与 [M4.md](M4.md) §6 一致）；**产出项也须全部为**归一后 **`defaultIncrement` 的正整数倍**。
- **`normalizeDefaultIncrement`**：将 `default_increment` **规范为整数 ≥1**（或团队约定最小步长）。
- **与 M4 对齐（步进网格，钉死）**：在得到归一 **`defaultIncrement = D`** 后，**`quickAmounts` 中每一项必须满足**：整数、**≥ D**、**且 `value % D === 0`**。非倍数：整库统一 **剔除** 或统一 **提升至「不小于原值的最小合法倍数** `Math.ceil(value / D) * D`」——**二者择一**，**禁止**下发仍非倍数之值。**禁止**仅把值 **round up 到 ≥ D** 却仍非倍数（例如 D=10 时把 15 留在响应里）。

`GET /api/materials` **必须**经此模块产出最终的 **`quickAmounts`** 与 **`defaultIncrement`**，使 M4 仅消费 **已归一** DTO。

## 前端：Materials 状态（轻量）

**不必上复杂状态库**；`useState` + `useEffect` / 自定义 hook 即可。

建议：

| 文件 | 职责 |
|------|------|
| `hooks/use-materials.ts` | 挂载时读缓存、静默请求、刷新、错误、`loading` |
| `lib/materials-client.ts` | `fetch('/api/materials', { credentials: 'include' })`、解析、`localStorage` 读写 key 统一 |

### `localStorage` 存储形状

```json
{
  "schemaVersion": 1,
  "generatedAt": "...",
  "materials": [ ... ]
}
```

- **`schemaVersion` 与服务器返回不一致**：**整块丢弃**，再写入新快照。  
- 成功拉取：**更新 React state + 上述 JSON**。

### 错误与离线（数据层）

- 请求失败：须进入下方 **§ UI 契约**中的 **错误条 + 重试**，不得只剩空白。
- **有上次成功缓存**：**继续展示旧列表**（Stale-while-error），但必须同时有 **置顶/醒目提示**（如「未能连接服务器，以下为上次更新」），避免用户以为仍为最新。
- **无缓存且首屏加载失败**：**禁止**白板；须有 **整块错误态** + **重试** + （若适用）跳转登录指引。

## UI 契约（M3 **强制**，禁止「半坏」状态）

以下四类必须 **有可读界面**，用户始终知道 **系统在干什么、自己能点什么**。

### 1) 刷新

| 要素 | UI 要求 |
|------|---------|
| 常驻入口 | **大号「刷新物料」**始终在物料相关页可视（或可一键滚达），不靠隐藏手势。 |
| 加载中 | 按钮 **`disabled` 或** 明确 **loading**（菊花/文案「更新中…」）；**防抖/禁连点**避免重复提交。 |
| 成功（静默拉取后也适用） | 可选短提示「已更新」；**强烈建议**展示 **本地化 `generatedAt`**（如「上次同步：今天 14:32」），便于用户判断是否为最新。 |
| 失败 | **独立错误区内**文案 + **重试**按钮；若在 **Stale** 模式下，错误区与缓存提示 **并存**，不得互相覆盖丢失。 |

### 2) 空状态

须区分 **`空数据`** vs **`加载中`** vs **`出错`**（三者 **不得**用一种空白糊弄过去）。

| 场景 | UI 要求 |
|------|---------|
| **HTTP 200** 且 `materials.length === 0` | 明确 **「暂无可用物料」** + 简短说明（如「请到 Airtable 将物料设为 Active」）+ **仍保留「刷新物料」**。 |
| **首启无缓存、正在请求** | **骨架屏 / Spinner /「正在载入物料…」** 任选其一或组合，勿与空列表态相同外观。 |
| **曾成功过但最后一条被 Active 清空** | 仍按 **空数据**模板展示，可加一句「已全部停用或暂无」避免误以为断网。 |

### 3) 401（未登录或会话失效）

| 要素 | UI 要求 |
|------|---------|
| 识别 | `GET /api/materials` **`401`** 时，前端 **不得**在无提示下静默失败或只转圈。 |
| 行为 | **立即**引导用户：**跳转 `/login`** 或在当前页 **全屏/条带**说明「请先登录」+ **前往登录**。 |
| 缓存 | **建议**清空 **仅服务端依赖的物料本地缓存键**（或整包重置），避免出现「看似已登录、列表冻结在旧快照」的假状态。**不得**只靠空白页表达。 |
| 登录返回后 | 用户回到物料页时应能 **自动或一键**重新拉取，无需硬刷新浏览器。 |

### 4) 缺省字段（服务端兜底 + 前端展示兜底）

服务端缺省遵守 §`GET /api/materials`；**前端还须**避免因 `null`/空串导致裂版：

| 字段 | UI 兜底 |
|------|---------|
| 主文案 | 展示 **`buttonLabel || name`**；若服务端已保证至少有 `name`，按钮 **不得空白**。 |
| `group` 空或未映射 | Chips 文案 **「未分组」**，或整块归入 **「其他」**。 |
| `category` 无法匹配 tab | **「其他」tab**（或归入第一个 tab）且 **单行可见**，不得隐藏行导致误以为没数据。 |
| `quickAmounts` 空 | **不展示快捷点**占位即可，**不出现**错误或裂口。 |
| `imageUrl` 为 `null` | **统一占位图标/色块**，**禁止**broken image。 |
| `unit`/`defaultIncrement` | 可在卡片角标轻量展示；缺失时服务端默认后，仍可显示 **`-`** 或与计划一致的默认单位短语（整库统一一句即可）。 |

需能验证 **显示逻辑**，不必接 **用途 / 数量 / 提交**：

- **`category` tabs**：Ingredient / Bottle / Label / Ribbon / Packaging（或与 `Materials.category` option 完全一致）  
- **`group` chips**：来自行的 **`group`**

- **`favorite`** 区域：可将 `favorite===true` 置顶块或专区（与排序一致即可）

## M3 验收标准

| # | 标准 |
|---|------|
| 1 | 未登录访问 `/api/materials` → **`401`** |
| 2 | 登录后能拿到 **`active`** 物料列表 |
| 3 | Browser **Network / bundle** **无** PAT |
| 4 | 响应 **仅含**约定的 PWA 字段；**无** supplier/price/notes/批次/raw 附件列表 |
| 5 | Airtable 将该行 **`active=false`** ，刷新后 **该项消失** |
| 6 | 改 **`favorite` / `sort_order` / `material_group`** ，刷新后 **UI 次序或分组随之变** |
| 7 | **`localStorage`** 有可读缓存（含 `schemaVersion`、`generatedAt`） |
| 8 | **断网或 Airtable 失败**：若有缓存→**仍展示旧列表** + **明确失败提示** + **重试** |
| 9 | **无搜索框**，仅靠分类/分组/列表可达多种 `category` 下的示例物料 |
| 10 | **数十～数百** active records 仍可接受性能（分页已实现） |
| 11 | **刷新**：有可见 **刷新物料**按钮；加载中禁用/明示 loading；失败后 **可读错误 + 重试** |
| 12 | **空状态**：能区分 **载入中 / 真·零条 / 错误**；零条时有说明 + 刷新 |
| 13 | **401**：会话失效时有 **跳转或前往登录**，并避免「冻结假列表」|
| 14 | **缺省字段**：无主文案裂口、无裂图，`group/category` 有 **「未分组」「其他」**等兜底文案 |
| 15 | **`quickAmounts` / `defaultIncrement`**：经 **`lib/quantity-defaults.ts`** 输出，**全为整数**；每项 quick **为用 `defaultIncrement` 规范化后的步长之正整数倍** |
